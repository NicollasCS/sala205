## рҹ”„ FLUXO DE UMA REQUISIГҮГғO

```
1. Cliente faz requisiГ§ГЈo
   GET /api/usuarios?admin-token=turma205-admin

2. Express recebe e passa por Middlewares
   в”ңв”Җ CORS Middleware вң…
   в”ңв”Җ JSON Parse вң…
   в””в”Җ Static Assets вң…

3. Router Principal (index.js) roteia
   /api/usuarios вҶ’ usuariosRoutes.js

4. Router EspecГӯfico (usuariosRoutes.js)
   GET / вҶ’ usuariosController.listUsers

5. Middleware de AutenticaГ§ГЈo (requireAdmin)
   в”ңв”Җ Verifica header x-admin-token
   в””в”Җ Se invГЎlido: retorna 403

6. Controller (usuariosController.listUsers)
   в”ңв”Җ Chama supabase.from('usuarios').select()
   в”ңв”Җ Usa helpers.normalizeUser()
   в”ңв”Җ Cria log com createLog()
   в””в”Җ Retorna res.json(users)

7. Response volta ao cliente
   200 OK com JSON
```

## рҹҺҜ SEPARAГҮГғO DE RESPONSABILIDADES

| Camada | Responsabilidade | Exemplo |
|--------|------------------|---------|
| **Config** | Constantes e configuraГ§Гөes | PORT, TOKENS, LIMITES |
| **Middleware** | ValidaГ§ГЈo e autorizaГ§ГЈo | isAdminToken(), requireAdmin() |
| **Utils** | FunГ§Гөes reutilizГЎveis | normalizeUser(), createLog() |
| **Controllers** | LГіgica de negГіcio | listUsers(), createEvento() |
| **Routes** | Mapeamento HTTP вҶ’ Controller | router.get('/') вҶ’ controller() |
| **Server** | Setup Express e listeners | Middleware global, PORT, listeners |
